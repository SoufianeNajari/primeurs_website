## Environment Variables

Copy `.env.local.example` to `.env.local` and fill in:

| Variable | Purpose |
|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Public anon key (browser queries) |
| `SUPABASE_SERVICE_ROLE_KEY` | Admin key (server-side only) |
| `RESEND_API_KEY` | Email delivery via Resend |
| `ADMIN_PASSWORD` | Plain-text password for `/admin` login |
| `SHOP_EMAIL` | Recipient for order notification emails |

## Architecture

This is a Next.js 14 App Router PWA for a local produce shop (Pontault Primeurs).

### Database (Supabase/PostgreSQL)

Two tables — see `seed.sql` for full schema:
- `produits`: product catalog (`id`, `nom`, `categorie`, `prix_kg`, `disponible`, `ordre`)
- `commandes`: orders (`id`, `client_nom`, `client_email`, `client_telephone`, `lignes` JSON, `message`, `statut`, `created_at`)

Two Supabase clients in `lib/supabase.ts`:
- **Public client** (`supabasePublic`) — uses anon key, safe for browser
- **Admin client** (`supabaseAdmin`) — uses service role key, server-side only

### State Management

`CartContext` (`components/CartContext.tsx`) is the single source of truth for cart state:
- Persisted to `localStorage` as `Record<string, CartItem>`
- On mount: fetches live product list, prunes unavailable items
- Subscribes to Supabase `postgres_changes` on `produits` — auto-removes items that go unavailable in real time
- Exposed via `useCart()` hook

### User Flow

1. **Browse** `/boutique` — SSR with `force-dynamic`, fetches products via admin client
2. **Cart** — `CartDrawer` slides in; items managed by `CartContext`
3. **Checkout** `/order` — `OrderForm` submits to `POST /api/order`
4. **Order API** — validates inputs, re-checks product availability, inserts into `commandes`, sends two emails (shop notification + customer confirmation) via Resend
5. **Confirmation** `/order/confirmation`

### Admin Panel

- `/admin` — login form sets cookie `admin_auth=true`; toggles product availability via `PATCH /api/toggle`
- `/admin/orders` — lists orders from last 7 days; updates status via `PATCH /api/orders/[id]`
- Auth is a simple cookie check in API routes — no middleware

### PWA

Configured via `next-pwa` in `next.config.js`:
- Service worker caches Supabase API responses (NetworkFirst, 5 min TTL) and images (CacheFirst, 1 day)
- Offline fallback: `/offline`
- Disabled in `NODE_ENV=development`
- Icons generated by `scripts/generate-icons.js`

### Email

`lib/mailer.ts` wraps Resend. On each order, two emails are sent:
- Shop owner (`SHOP_EMAIL`): order details
- Customer (`client_email`): confirmation receipt
